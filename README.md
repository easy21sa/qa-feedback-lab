# QA Feedback Lab

A small public project focused on structured QA feedback and reproducible bug reporting.

## Overview
This demo was created to practice documenting product issues in a clearer and more useful way. It shows a simple front-end page with sample bug reports, severity labels, expected vs actual behavior, and reproduction steps.

## What it includes
- structured bug reporting examples
- severity classification
- expected vs actual result format
- reproducible testing steps
- simple static front-end built with HTML and CSS

## Why I built this
I’m using small public projects like this to improve how I document issues, think through product feedback, and build in public more consistently.
